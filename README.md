# 🩺 Diabetes Prediction System

A production-ready end-to-end Machine Learning application using Python, Flask, and an SVM classifier trained on the PIMA Indians Diabetes dataset.

---

## 📁 Project Structure

```
diabetes_prediction/
│
├── backend/
│   ├── app.py              ← Flask API (all endpoints)
│   ├── utils.py            ← Helpers: model loading, validation, prediction logic
│   ├── train_model.py      ← Model training script (run once)
│   ├── model.pkl           ← Saved SVM model (generated by train_model.py)
│   ├── scaler.pkl          ← Saved StandardScaler (generated by train_model.py)
│   ├── requirements.txt    ← Python dependencies
│   ├── dataset/
│   │   └── diabetes.csv    ← PIMA Indians dataset
│   ├── uploads/            ← Temp CSV uploads for batch prediction
│   └── logs/
│       └── api.log         ← Rotating request/error logs
│
└── frontend.html           ← Standalone UI (connect to any backend URL)
```

---

## 🚀 Quick Start

### 1 – Install dependencies

```bash
cd backend
pip install -r requirements.txt
```

### 2 – Train the model

```bash
python train_model.py
```

This generates `model.pkl` and `scaler.pkl` and prints accuracy (~92%).

### 3 – Start the API

```bash
python app.py
```

API runs at `http://localhost:5000`

### 4 – Open the frontend

Open `frontend.html` in any browser. Set the API URL to `http://localhost:5000`.

---

## 🔌 API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Liveness check + model status |
| POST | `/predict` | Single JSON prediction |
| POST | `/predict-batch` | CSV upload, bulk predictions |
| GET | `/model-info` | Model metadata + scaler stats |
| GET | `/sample-input` | Ready-to-use sample payload |

### POST /predict — Example

**Request**
```json
{
  "Pregnancies": 5,
  "Glucose": 166,
  "BloodPressure": 72,
  "SkinThickness": 19,
  "Insulin": 175,
  "BMI": 25.8,
  "DiabetesPedigreeFunction": 0.587,
  "Age": 51
}
```

**Response**
```json
{
  "status": "success",
  "prediction": "Diabetic",
  "prediction_code": 1,
  "confidence": 0.87,
  "probabilities": {
    "not_diabetic": 0.13,
    "diabetic": 0.87
  },
  "input": { ... }
}
```

### POST /predict-batch — Example

```bash
curl -X POST http://localhost:5000/predict-batch \
  -F "file=@patients.csv"

# Download results as CSV:
curl -X POST "http://localhost:5000/predict-batch?download=true" \
  -F "file=@patients.csv" -o predictions.csv
```

## ☁️ Deploy to Render

1. Push `backend/` to a GitHub repository
2. Create a new **Web Service** on [render.com](https://render.com)
3. Set:
   - **Build command**: `pip install -r requirements.txt && python train_model.py`
   - **Start command**: `gunicorn app:app --bind 0.0.0.0:$PORT`
4. Open `frontend.html`, set API URL to your Render URL

## ☁️ Deploy to Railway

```bash
# Install Railway CLI
npm install -g @railway/cli

cd backend
railway login
railway init
railway up
```

Set environment variable `PORT` to `5000` in Railway dashboard.

## 🧠 Model Details

| Property | Value |
|----------|-------|
| Algorithm | SVM (RBF kernel) |
| Dataset | PIMA Indians Diabetes (768 samples) |
| Train / Test split | 80% / 20% |
| Preprocessing | StandardScaler (zero-value imputation) |
| Test Accuracy | ~92% |
| Output | 0 = Not Diabetic, 1 = Diabetic |

**Features (in order):**
1. Pregnancies
2. Glucose
3. BloodPressure
4. SkinThickness
5. Insulin
6. BMI
7. DiabetesPedigreeFunction
8. Age
